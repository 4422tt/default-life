# 预制人生 Default Life

一个本地优先的个人默认选择系统。先维护自己认可的选择池，再根据预算、精力、天气、同行人和时间等情境，得到可解释的推荐。

**在线体验：** [https://4422tt.github.io/default-life/](https://4422tt.github.io/default-life/)

## 当前 MVP

- 默认池：新增、编辑、删除、启用、暂停和标记最近想吃
- 情境输入：预算、精力、天气、同行人、熟悉或探索、时间压力
- 推荐结果：一个主推荐、两个备选、手动选择和真实评分解释
- 反馈闭环：很准、还行、暂时别推荐，后者会进入 14 天冷却期
- 历史记录：保留当时情境、推荐结果、实际选择和反馈
- 本地数据：Dexie 与 IndexedDB，无账号、无服务端依赖
- 数据可控：JSON 导出、恢复和重置示例
- 界面：桌面侧栏、手机底部导航、明亮、深色和系统主题
- 生活导入：真实识别接口、多图上传、结果编辑、错误处理、确认后写入和生活画像
- 人生轨迹：把选择次数、反馈变化和生活导入整理为时间线

## 推荐规则

推荐先执行硬过滤，再计算可解释分数：

- 个人喜欢程度：35 分
- 当前情境匹配：25 分
- 最近是否重复：15 分
- 历史反馈修正：15 分
- 最近想吃：10 分

“吃熟悉的”会选择最高分结果；“换换口味”会在前五名中按分数加权抽取。预算没有可用选项时会放宽预算条件，并在解释里明确说明。

## 本地运行

```bash
pnpm install
pnpm dev
```

打开 `http://localhost:3000`。

## 工程校验

```bash
pnpm typecheck
pnpm test
pnpm build
```

生产构建使用 Next.js 静态导出，生成目录为 `out`。

## 目录

- `app/`：页面入口、主题和全局视觉系统
- `components/`：主流程、默认池和历史界面
- `lib/recommendation.ts`：过滤、评分、抽样和解释
- `lib/order-recognition.ts`：真实订单识别接口调用与文件校验
- `lib/order-normalization.ts`：订单规范化、去重与偏好摘要
- `ORDER_RECOGNITION.md`：服务端接口和 API Key 配置说明
- `lib/storage.ts`：IndexedDB 写入、反馈和备份
- `lib/types.ts`：领域模型
- `tests/`：推荐规则单元测试
